-내용 정리

 객체 지향이랑 영어로 'Object Oriented'라 하고 "위주" 혹은 "선호"라고 하는 것 이 어울린다. 객체를 선호하고 위주로 한다고 하는 것이 어울린다. 그래서 '객체 위주'라고 이해하는 것이 편하다.

 객체지향에는 중요한 키워드 5가지가 있다. 클래스, 오브젝트, 캡슐화, 상속성, 다형성이다.

 

클래스(class) = 추상 (abstacrt)

클래스는 표현 대상의 특징에 대한 서술을 말한다  //공통 특징, 서술

 

 오브젝트(object)= 실체(instance)

오브젝트는 클래스의 인스턴스이다. //구체, 실체 존재 , 고유성

오브젝트는 데이터와 데이터에 대한 조작의 묶음이다.

오브젝트의 모양은 클래스(class)에 서술되어 있다.

 

다양한 용어 그러나 같은 뜻

attribut = data =member variables = state =field

behavior = operation = member funcion =method

클래스(class) = 개념(concept) = 타입(type)

객체(object) = 실체(instacne) = 변수(variable)

 

 

 캡슐화,상속성,다형성 이 세가지 특성을 지원하는 언어를 객체지향 프로그래밍 언어라고 한다.

캡슐화는 데이터와 데이터를 다루는 방법을 묶는 것이다.

data + operation on data

 상속성은 하나의 클래스가지고 있는 특징들을 그대로 다른 클래스가 물려 받는 것이다.

IS-A로 상속 관계를 표현합니다. 재사용, 유연성이라는 특징을 가진다.

 다형성 상속를 따라서 각각의 클래스에 한가지 이름을 줄 수 있는것.

간단히 말해, 같은 명령을 각기 다른 오브젝트에 줄 수 있다는 뜻

 

 객체지향은 프로그래밍 패러다임 중의 하나입다.

자바와,C++가 객체지향 프로그래밍을 사용한다.

최초의 객체 지향 언어는 "Simula67"이다. 시뮬레이션을 위해 만들어진 언어이다.

 객체지향의 장점은 자연스런 모델링, 생산성 향상, 점진적 개발, 견고한 의미적 모델링, 명확한 인터페이스 기술, 보안 ,유지보수의 용이성 등이 있다.

객체지향 프로그래밍은 객체와 객체간 역할과 관계 중심, 객체 중심의 특성을 지닌다.

구조적 프로그래밍은 기능적 분할 절차 중심의 특성을 가진다.

 

 느낌점:객체지향이라는 평소에 내가 어렵게 느끼던 개념을 쉽게 예시와 특징에 대해 알려줘서 이해하는데에 도움이 됐다.

 커맨드 라인은 command-line interface, cmd, CLI, 프롬프트, 콘솔, 터미널 라고도 불리는 텍스트 기반 응용 프로그램으로 파일보기, 처리 조작이 가능하다. 윈도우 익스플로러나 맥 os의 파인더와 비슷하지만 그래픽 인터페이스가 없다.

 

 커맨드 라인 열기

윈도우:시작메뉴 ->  모든프로그램  -> 보조프로그램 -> 명령 프롬프트

Mac os X: 응용 프로그램 -> 유틸리티 -> 터미널

리눅스: 프로그램 -> 보조 프로그램-> 터미널

 

 명령어

whoami:  컴퓨터의 사용자 이름을 스크린에 보여준다.

cd: 현재 어느 디렉터리에 있는지 확인

dir: 현재 위치한 디렉터리에 파일과 디렉터리 목록 보기

cd <A>: 현재 디렉터리에 A로 이동한다 //만약 A의 이름이 desktop일때 d만치고 tap을 눌러 자동완성을 할 수 있다.

mkdir: 현재 디렉터리에 새로운 디렉터리를 만든다.

cd.. : 상위 디렉터리로 이동한다.

rmdir : 파일 삭제

exit: cmd창 종료
